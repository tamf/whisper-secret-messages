Frequently Asked Questions

On the Secret Messages page, enter your message into the text box.

You may also choose to specify additional pieces of information: (1) the passphrase, which is used for decrypting the message, (2) the access limit, which specifies the number of times your message can be retrieved before it is deleted from the database, and (3) the time expiry. Please note that if the optional information is not provided, the program will use the default values as inputs. The default for access limit is five times and the default for time expiry is one hour. If a passphrase is not provided, it will be randomly generated.

After clicking send, your message will first be encrypted before it is stored into our Cloud Firestore database. A link which you can share with your friends will then be generated so they can access your secret!

Once the receiving user accesses the shareable link, they will be redirected to the Retrieve Secret page. The required information will auto-fill so all they need to do is click the retrieve button to access your message. However, if a passphrase was provided in step 1, the receiving user will also need to manually input it into the passphrase box.

The following encryption steps were performed using Web Crypto, a cryptography API available in modern web browsers:

(1) We first create a password based key using PBKDF2, a key derivation function that applies hash functions to the inputted passphrase to derive a secret key. This key will be used to derive our encryption key for encryption and decryption.

(2) We use the above-derived key as input to generate the AES-GCM encryption key. In this stage, we also specify the salt, the initialization vector (iv), the hash algorithm we will use on the PBKDF2 key (we will be using SHA-256), and the number of iterations in which the hash algorithm will be applied to the PBKDF2 key. A salt is randomly generated data that gets appended to the PBKDF2 key, before the hash algorithm is applied, to add an additional layer of security. The iv is randomly generated data that is used to prevent repetition in data encryption.

(3) Finally, the AES-GCM key is used to encrypt your message. You can think of encryption as the act of scrambling the contents of your message.

Your secret message is encrypted before it is stored into our database so even if a hacker had full access to our database, they would not be able to decipher the messages. Furthermore, the passphrase entered in step 1 is used for decrypting the secret message. Therefore, even if someone intercepts the generated link, they would not be able to decrypt the secret message unless they also possessed the passphrase.
